# sqb run

Same as `build` but skips SQL unit tests and audits. Useful for fast iteration when you only want to materialize models.

## Usage

```bash
sqb run --project-dir <path> [flags]
```

## Flags

Same as `build`:

| Flag | Description |
|------|-------------|
| `--project-dir` | Path to the project directory |
| `--no-color` | Disable ANSI color output |
| `--no-sql-validation` | Skip compile-time SQL syntax validation |
| `--fail-fast` | Stop on first failure |
| `--full-refresh` | Full rebuild of selected models |
| `--concurrency` | Number of worker connections |
| `--select` | Select specific models |
| `--exclude` | Exclude specific models |
| `--verbose` | Show lifecycle SQL inline |
| `--debug` | Enable debug diagnostics |

## Differences from build

| Feature | build | run |
|---------|-------|-----|
| Seeds | Yes | Yes |
| SQL unit tests | Yes | No |
| Model materialization | Yes | Yes |
| Audits | Yes | No |
